## Goal

Add thorough rustdoc comments to every public item in `branch.rs` and `worktree.rs`. These two types are clean and well-defined — document them to that standard.

## What to document

For each public struct, method, and trait impl:
- What it does
- When/why it panics (Branch::new panics on invalid git ref names via git2::Reference::is_valid_name)
- When it returns an error and which error variant
- For WorktreeRef vs Worktree: explain the distinction (serializable path reference vs validated live handle)
- For Worktree: explain the guarantee (underlying Repo is a linked worktree, not the main clone)

Module-level doc comments should give an overview of the type and its role in the crate.

## Acceptance
- [ ] Every public item has a doc comment
- [ ] Panics are documented with `# Panics` sections
- [ ] Errors are documented with `# Errors` sections where applicable
- [ ] `cargo doc` builds without warnings
- [ ] No code changes — docs only

## Out of scope
- Changing any behavior or signatures
- Documenting repo.rs or other files
