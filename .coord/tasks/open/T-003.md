## Goal
Replace `Command::new("git")` calls in `git/mod.rs` tests with raw git2 API calls.

The test helpers `init_repo`, `add_self_remote`, `create_bare_upstream` and inline git CLI calls in each test should use git2 directly: `git2::Repository::init()`, `repo.commit()`, `repo.branch()`, `repo.remote()`, etc.

Use raw git2 for test setup, NOT our `Repo` wrapper — keeps tests independent of the code under test.

## Acceptance
- [ ] No `Command::new("git")` in `git/mod.rs`
- [ ] All existing tests still pass with same semantics
- [ ] `cargo test -p jig-core` passes
