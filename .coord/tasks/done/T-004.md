## Goal

Refactor `crates/jig-core/src/github/mod.rs` to use the same client+request-per-file pattern as the Linear provider refactor (T-001). Break the module into a client with separate request files for each routine (PR info, CI checks, reviews, etc.).

## Acceptance
- [x] GitHub client struct with typed request pattern (like LinearProvider)
- [x] Each major operation (PR status, CI checks, reviews, commits, conflicts) in its own file
- [x] Public API surface unchanged or improved
- [x] Builds cleanly

## Out of scope
- Changing the underlying `gh` CLI integration
- Adding new GitHub features
- Touching non-github modules

- Introduced GitHubError as module-owned error type with #[from] in crate::error::Error
